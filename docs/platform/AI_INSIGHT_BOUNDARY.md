# AI_INSIGHT_BOUNDARY

## Why AI Insight Boundary Exists

RepoSense keeps detection and evidence processing transparent in OSS while allowing room for hosted/commercial enhancement layers.

This boundary prevents confusion between:

- conservative, evidence-backed local outputs
- deeper guided collaboration products built on top

## OSS Layer: What This Repository Provides

The open-source layer provides usable local insight workflows:

- evidence-backed signals from Facts
- backend verifier report
- deterministic patterns
- constrained local `ai summary` / `ai risks` / `ai explain` / `ai ask`
- context pack and run-manifest based handoff

These outputs are conservative and reproducible.

## Hosted / Commercial Layer: What May Be Enhanced

Hosted/commercial layers may provide enhancement features such as:

- guided explanation enrichment
- repair suggestions and playbooks
- prioritized upgrade advice
- team/project history
- cross-run trend analysis
- enterprise/compliance reporting

## Boundary Statement

AI Insight is not included in the open-source engine.

This repository provides evidence-backed signals and conservative local reports only.

Guided explanation enhancement, repair suggestions, multi-run history, and enterprise reporting may be provided in hosted/commercial layers.

## Why detector-open / deep-insight-reserved is reasonable

- Open detector and evidence logic builds trust and verifiability.
- OSS users can run end-to-end local analysis without cloud dependency.
- Enhanced collaboration and long-horizon insight orchestration can evolve in hosted/commercial offerings.

## Facts first, source on demand still applies

AI insight boundaries do not change core principles:

- Facts first
- Source on demand
- Evidence-first
- Deterministic and reproducible outputs

## Important Clarification

RepoSense OSS is independently usable.

Hosted/commercial layers are enhancements for collaboration depth and productivity, not a requirement for core verifier value.
