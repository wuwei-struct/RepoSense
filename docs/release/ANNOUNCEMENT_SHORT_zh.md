# RepoSense 对外发布文案（短版）

## 版本 1：一句话版（<= 40 字）

RepoSense 是面向 AI 辅助开发项目的后端事务、副作用与升级上下文系统。

## 版本 2：短版（约 100–150 字）

RepoSense 面向 AI 辅助开发项目，把仓库从源码堆转成 `Code -> Facts -> Patterns -> Insights` 的可追溯链路，并进一步提供升级上下文交接能力。它重点暴露后端事务、副作用、队列、缓存与风险信号，默认 Facts-first、Evidence-first，必要时才 source-on-demand 下钻。当前已支持 Analysis、Learn、AI Summary/Risks/Explain/Ask（受限）与本地 Demo 闭环。

## 版本 3：中版（约 250–400 字）

如果你已经能用 AI 很快生成后端第一版，下一步真正的难题通常不是“写出来”，而是“改得稳、交得清”。  
RepoSense 的定位是：面向 AI 辅助开发项目的后端事务、副作用与升级上下文系统。它先把代码仓库转成可审计工程事实（Findings / Events / Evidence / Event Graph），再生成确定性模式（Patterns）和 grounded 洞察（Insights），并通过 Context Pack、Baseline & Diff、Run Manifest 把下一轮升级所需上下文交接出来。  

它不是默认让 AI 漫游整仓源码，也不是完整正确性证明器。RepoSense 强调 Evidence-first、Deterministic、Facts first, source on demand：先看事实，再按证据受限下钻。对团队而言，这意味着不仅能看清后端事务和副作用信号，还能在 AI 辅助维护与升级时，获得可复现、可交接、可审计的上下文基础。
