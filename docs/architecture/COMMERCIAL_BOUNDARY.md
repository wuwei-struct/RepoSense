# Commercial Boundary

## Open-core Architecture

RepoSense uses an open-core split:

- OSS layer: fact extraction, deterministic patterning, local verification workflows
- Cloud/AI layer: higher-order orchestration and enterprise delivery

## OSS Layer

The OSS layer includes:

- Scanner and Fact Engine
- Rulesets and pattern detection logic
- CLI/CI workflows
- Event, Evidence, and Finding schemas
- Local report and learn surfaces
- Context Pack + SARIF exports

## Cloud / AI Layer

The service layer can include:

- Managed run history and team coordination
- Enterprise workflow templates and policy packs
- Premium explain/prioritize/repair orchestration
- Long-horizon upgrade advisory

## Allowed OSS Outputs

- Findings / Events / Evidence
- report and graph artifacts
- patterns and deterministic summaries
- local learn and local insight outputs
- gate / baseline / manifest artifacts

## Reserved Service Outputs

- Managed multi-run intelligence across organizations
- Team-level collaboration memory and governance
- Premium explain/repair playbooks and workflow automation
- Enterprise-tailored reporting packages

## Extension Points

- Custom rulesets
- Custom pattern post-processing
- Custom CI gating pipelines
- Custom report consumption and data integration

## Why Detection Open + Explanation Enhancements Reserved

Open detection logic ensures trust, reproducibility, and ecosystem adoption.
Reserved explanation/repair enhancements allow sustainable investment in advanced service capabilities without obscuring core verifier behavior.
