# Open Source Strategy

## Principle

Open-source what makes RepoSense immediately usable.
Provide service layers for capabilities that create long-term lock-in value.

## Open-Source Components

- Fact Engine / Scanner
- Pattern matching logic
- Rulesets
- CLI / CI Runner
- Context Pack spec
- SARIF export
- Local Learn UI
- Event / Evidence / Finding schema

## Reserved Components

- AI Insight prompt engineering
- Cloud run history
- Team collaboration features
- AI Q&A context management services
- Enterprise report templates
- Advanced repair suggestions
- Long-term upgrade advisor

## Special Boundary

Pattern detection logic remains open-source.
Pattern explanation playbooks, repair playbooks, and premium AI Insight enhancements can remain reserved service layers.

## Why This Boundary Works

- Detection transparency builds trust and adoption.
- Open schemas and outputs enable ecosystem integrations.
- Reserved interpretation and orchestration layers sustain commercial product development.
